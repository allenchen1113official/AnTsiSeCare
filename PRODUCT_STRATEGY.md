### 2.3 「正常 / 異常」的非文字判斷流程

```
用戶點選某圖示後：

情境 A：正常狀態
  → 圖示底色變綠 ✓（不需要額外說明）

情境 B：異常狀態
  → 圖示底色變紅 ✗
  → 彈出「嚴重程度」選擇（3個表情符號）：
      😐 輕微  |  😟 需注意  |  🚨 緊急
  → 「緊急」選項觸發自動通知家屬

情境 C：數值型（血壓、體溫、血糖）
  → 數字鍵盤輸入（數字無語言障礙）
  → 系統自動判斷是否在正常範圍
  → 超標自動標紅並通知
```

### 2.4 語音轉文字對應中文資料

```dart
// 語音記錄流程：選語言 → 錄音 → 轉文字 → 自動翻中文

class VoiceNoteService {
  // step1: 使用 speech_to_text package 轉換
  Future<String> speechToText(String langCode) async {
    // langCode: 'id-ID', 'vi-VN', 'th-TH', 'fil-PH', 'zh-TW'
    final result = await SpeechToText().listen(localeId: langCode);
    return result.recognizedWords;
  }
  
  // step2: 呼叫 Claude API 翻譯成中文（保留長照術語）
  Future<String> translateToZhTW(String text, String fromLang) async {
    // 離線模式：使用本地術語對照表
    final localResult = glossary.lookup(text, fromLang);
    if (localResult != null) return localResult;
    
    // 有網路：呼叫 Claude API
    return await claudeTranslate(text, fromLang, 'zh-TW');
  }
}
```

## 四、零成本營運策略

| Firestore 讀取 | 50,000 次/天 | 500 位日活用戶 × 100 次讀取 |

```

```

### 4.4 機構資料自動更新策略（零伺服器費）

```
策略：每次 APP 啟動時，檢查本地快取是否超過 24 小時
↓
若超過 → 前端直接 fetch 政府 CSV（無後端）
↓
解析後存入 Hive 本地 + 上傳 Firestore 共享快取
↓
其他使用者直接讀取 Firestore（不再重複 fetch 政府 API）

效益：
  - 政府 API：每日僅被 fetch 1 次（第一位開啟的用戶）
  - Firestore 讀取：其餘用戶讀 Firestore 快取，省 API 配額
  - 無需後端 Cron Job，完全無伺服器
```
