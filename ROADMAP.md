# AnTsiSeCare - 開發路線圖

---

## 一、整體開發階段

```
Phase 0：需求確認與設計（第 1-2 月）
Phase 1：MVP 核心功能（第 3-6 月）
Phase 2：完整功能擴充（第 7-10 月）
Phase 3：優化與推廣（第 11-12 月）
Phase 4：進階功能（第 2 年）
```

---

#### Sprint 3（第 5 月）：照護核心 + 雙語日誌

## 六、風險管理

| 風險 | 可能性 | 影響 | 對策 |
|------|--------|------|------|
| 長者使用障礙 | 高 | 高 | 強化長者模式、到府教學 |
| 機構資料不完整 | 中 | 中 | 與縣府合作取得官方資料 |
| 個資外洩 | 低 | 極高 | 定期資安稽核、加密儲存 |
| App Store 審核退件 | 中 | 中 | 提前了解審核規範，保留2週緩衝 |
| 照服員抗拒數位化 | 中 | 高 | 簡化操作流程、辦理教育訓練 |
| 偏鄉網路不穩定 | 高 | 中 | 離線模式、SMS 備援 |
| **印尼語翻譯品質不佳** | **中** | **高** | **人工校對機制、長照術語固定詞彙表** |
| **移工隱私疑慮** | **中** | **高** | **清楚告知蒐集範圍、印尼語隱私政策** |
| **文化差異造成誤解** | **低** | **中** | **聘用在台印尼移工參與 UX 測試** |

## 七、完整 Prompt 指引

以下為向 AI 開發工具輸入的完整系統 Prompt，可用於啟動 AnTsiSeCare 的程式開發：

```
你是一位資深全端工程師，正在協助開發一款名為「AnTsiSeCare（安心照護）」的台灣長照 APP。

【專案背景】
- 以苗栗縣為場域，整合苗栗縣 18 個鄉鎮市的長照 A/B/C 級服務資源
- 使用者包括：台灣長者、家庭照顧者、本籍照服員、印尼籍家庭看護工（第一優先）、照管專員、機構管理員
- 多語言支援：繁體中文（主）、印尼語 Bahasa Indonesia（P1）、英語（P2）
- 需支援客語（四縣腔）語音介面、長者大字體模式、偏鄉離線功能
- 照護日誌需支援印尼語填寫後自動翻譯成中文供家屬閱讀

【技術棧】
- 前端 APP：React Native 0.74（TypeScript）
- 多語言：i18next + react-i18next + react-native-localize
- 狀態管理：Zustand + React Query
- 後端：NestJS（Node.js TypeScript）
- 資料庫：PostgreSQL + Prisma ORM
- 快取：Redis
- 推播：Firebase Cloud Messaging（多語言 payload）
- 地圖：Google Maps API
- 翻譯服務：Claude API（claude-sonnet-4-6，長照術語翻譯）
- 部署：Docker + AWS ECS / Zeabur

【設計規範】
- 主色：#2E7D6B（綠色系，信任感）
- 緊急色：#E53935（紅色，SOS）
- 最小字體：16px，長者模式 20px
- 觸控目標：最小 48dp
- 符合 WCAG 2.1 AA 無障礙標準
- 圖示優先設計（減少語言文字依賴，對外籍看護友善）

【語言切換邏輯】
- 裝置語言為印尼語（id-ID）→ 自動切換印尼語介面
- 使用者可在設定手動切換語言
- 翻譯缺漏時 fallback 至繁體中文

【核心功能優先順序】
1. 手機 OTP 登入 + 角色系統（含語言偏好設定）
2. 苗栗縣長照資源地圖（雙語：中文/印尼語）
3. 服務預約系統（含客語/印尼語照服員配對）
4. 電子照護日誌（印尼語填寫 → 自動翻譯中文）
5. SOS 緊急通報（含印尼語語音 + 1955 移工專線）
6. 健康數值記錄（數字/圖示為主，語言無關）
7. 用藥提醒（印尼語藥名對照）
8. 費用管理與補助試算
9. 雙語照護詞彙學習模組

請依據上述規格，協助我逐步實作 AnTsiSeCare APP，優先完成多語言（i18n）架構與印尼語照護日誌功能。
```
